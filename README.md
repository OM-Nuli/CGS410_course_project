# CGS410 Course Project — Dependency Length Minimization in LLMs

## Research Question

Do large language models (LLMs) generate sentences that obey **Dependency Length Minimization (DLM)** like human languages?

Human languages are known to optimize sentence structure by keeping syntactically related words close together, reducing cognitive load and improving processing efficiency. Since LLMs are trained purely on next-word prediction with no explicit structural constraints, this raises an important question: *do they naturally learn and reproduce DLM?*

---

## Methodology

For each sentence, dependency length is computed as the linear distance between a word and its syntactic head:

```
Distance = |position of dependent − position of head|
```

Three types of data are compared:

**1. Human Language Data** — Gold-standard UD treebank sentences, cleaned (punctuation removed), and re-parsed through Stanza to get dependency lengths.

**2. LLM-Generated Sentences** — Sentences generated by two LLMs (Grok and Qwen), parsed through Stanza using the same pipeline as human data. Two separate sets of outputs are maintained for each model.

**3. Random Baseline** — Human and LLM sentences with words randomly shuffled, then re-parsed. This establishes an upper bound — a worst-case dependency length with no structural optimization.

## Pipeline Overview

### 1. Human Parser (`Human/ud_parser.py`)

Reads `.conllu` files from the UD treebanks, strips punctuation tokens, then re-parses cleaned sentences through Stanza to compute dependency lengths.

```
Input:  Human/Data_ud/<language>/*.conllu
Output: Human/outputs/<language>_human.json
```

### 2. LLM Parser (`LLM/llm_parser.py`)

Reads LLM-generated sentence JSON files and parses them through Stanza using the same pipeline as the human parser.

```
Input:  LLM/LLM_sentences_grok/<lang>_llm_sentences.json
Output: LLM/output_grok/<language>_llm.json
```

For Qwen sentences (CSV format), use `LLM_sentence_generation_Qwen.py` to generate, then run the parser against `LLM_sentences_Qwen/`.

### 3. Random Baseline (`Random/random_parser_*.py`)

Shuffles words within each sentence randomly, then re-parses through Stanza. This creates a linguistically meaningless upper bound for dependency length. Four variants exist covering human and LLM (Grok/Qwen) inputs.

```
Input:  UD .conllu files  OR  LLM sentence JSON files
Output: Random/Random_Human2/ | Random_Grok2/ | Random_Qwen2/
```

> **Note:** For Chinese and Thai (no whitespace between words), the random parsers tokenize by character before shuffling.

---

## Output Format

All parsers produce JSON files in this format:

```json
[
  {
    "sentence": "The dog runs in the park",
    "avg_dep_length": 1.6,
    "sentence_length": 6
  }
]
```

Sentences shorter than 3 words are filtered out across all parsers.

---

## Processing Details

- Sentences are processed in **batches of 64** using `stanza.bulk_process()`
- Stanza models are **cached** after first load to avoid reloading between languages
- ROOT dependencies (head = 0) are excluded from all dependency length calculations
- Model loading is slow on first run — this is expected

## Expected Results

| Condition         | Expected Avg Dependency Length |
|-------------------|-------------------------------|
| Human sentences   | Lowest (DLM optimized)        |
| LLM sentences     | Similar to human, possibly slightly higher |
| Random baseline   | Highest (no structural optimization) |
